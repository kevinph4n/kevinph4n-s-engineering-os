# Engineering Playbook

This repository is a collection of lessons learned from hackathons, research projects, and engineering competitions.

I started building it after my first robotics hackathon, where my team finished 4th out of 10 teams. We did well, but I realized that engineering isn't just about writing code. It's also about building systems that help you solve problems repeatedly.

Instead of relying on memory after every event, I'll document:
- What worked.
- What failed.
- Why it failed.
- How to prevent it next time.

The goal isn't to become better at one hackathon.

The goal is to become a better engineer.
